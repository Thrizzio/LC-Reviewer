Given what you described, the implementation can stay fairly small. The goal is simply:

show today’s new problems

show today’s reviews

allow mark solved / not solved

reschedule if unsolved

Below is a practical step-by-step plan.

1. Define the Core Behavior

Your system will follow these rules:

Inputs

list of LeetCode problem IDs

newPerDay

reviewAfterDays

Daily logic

Assign newPerDay new problems.

Show problems whose reviewDate == today.

If a new problem is not solved, move it to tomorrow’s new list.

If solved → schedule review.
